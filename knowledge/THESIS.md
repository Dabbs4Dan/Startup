## How this file works

- **One thesis at a time.** This is not a list of ideas — it's the current bet.
- **Currently exploring** = active threads where the founder is still gathering signal.
- **Decided** = commitments. The agent treats these as constraints, not options to relitigate.
- **Ruled out** = ideas the founder has explicitly killed. The agent should not resurrect them without new evidence.
- **Update trigger:** any time the founder commits to a thesis, rules an idea out, or shifts direction — update in the same turn. Don't wait for `/end-session`.
- **Reasoning lives next to the decision.** Every entry has a 1-line "Why" so future-me knows whether new evidence should overturn it.
